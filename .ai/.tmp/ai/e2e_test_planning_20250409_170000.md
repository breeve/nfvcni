# nfvcni E2E 测试规划方案

## 1. 项目概述

nfvcni 是一个基于 eBPF 的网络虚拟化 CNI 插件，采用控制面+数据面分离架构：

- **控制面** (`pkg/controlplane`): REST API 管理接口，支持动态配置管理
- **数据面** (`pkg/dataplane`): eBPF 实现 + gRPC 服务

参考 Kube-OVN 的 E2E 测试方法，使用 Kind 构建 Kubernetes 测试集群，使用 Ginkgo 编写测试用例。

---

## 2. 测试需求分析

### 2.1 现有功能

| 组件 | 功能 | 接口 |
|------|------|------|
| 控制面 | Port 配置管理 | REST API `/admin/ports` |
| 控制面 | FDB 表查询 | REST API `/admin/fdb` |
| 控制面 | 健康检查 | REST API `/health` |
| 数据面 | Port 挂载 | gRPC `AttachPort` |
| 数据面 | Port 卸载 | gRPC `DetachPort` |
| 数据面 | FDB 条目管理 | gRPC `AddFdbEntry`, `DelFdbEntry`, `GetFdbTable` |
| 数据面 | Port 配置查询 | gRPC `GetPortConfig`, `UpdatePortConfig` |

### 2.2 测试场景

1. **基础功能测试**
   - 控制面服务启动/停止
   - 数据面服务启动/停止
   - 健康检查接口
   - API 认证 (可选)

2. **Port 配置管理测试**
   - 添加 Port (L2 mode)
   - 添加 Port (L3 mode)
   - 添加 Port (VlanIF mode)
   - 删除 Port
   - 更新 Port 配置
   - 查询 Port 列表
   - 查询单个 Port 配置

3. **FDB 表管理测试**
   - 添加 FDB 条目
   - 删除 FDB 条目
   - 查询 FDB 表
   - FDB 老化测试 (可选)

4. **数据面 gRPC 测试**
   - AttachPort/DetachPort
   - gRPC 服务的正确性

5. **集成测试**
   - 控制面配置变化同步到数据面
   - 高并发配置更新 (可选)
   - 故障恢复测试 (可选)

## 3. 可行性分析

### 3.1 依赖工具

| 工具 | 版本要求 | 用途 |
|------|----------|------|
| Go | >= 1.21 | 开发语言 |
| Kind | latest | 本地 Kubernetes 集群 |
| Ginkgo | latest | 测试框架 |
| Docker | latest | 容器运行时 |
| kubectl | latest | Kubernetes 客户端 |

### 3.2 技术方案

**方案 A: 独立容器测试 (推荐)**
- 在 Kind 集群中运行控制面和数据面容器
- 使用 Ginkgo 测试用例通过 Service/Pod 进行验证
- 优点: 环境真实，可测试 CNI 插件实际场景
- 缺点: 需要制作包含 eBPF 依赖的容器镜像

**方案 B: 本地 e2e 测试**
- 在测试节点上直接运行控制面和数据面进程
- 使用 Ginkgo 直接调用 REST API 和 gRPC
- 优点: 简单快速，不需要容器镜像
- 缺点: 无法验证 CNI 插件完整场景

**方案 C: 混合模式**
- 先实现方案 B 作为基础测试
- 后续实现方案 A 进行完整 CNI 验证
- 渐进式完善

### 3.3 风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| eBPF 需要高内核版本 | 测试环境受限 | 使用兼容内核的 CI/CD 容器或本地测试 |
| gRPC 端口映射依赖 | 网络配置复杂 | 使用 Service 暴露或 Host 网络模式 |
| KIND 不支持 eBPF | 无法在 Kind 中运行 eBPF | 跳过 eBPF 特定测试或使用特权容器 |
| 测试环境搭建复杂 | 开发体验差 | 提供一键搭建脚本 |

### 3.4 可行性结论

**可行性: 中等**

- 基础 REST API 测试完全可行
- gRPC 测试可行，需要解决服务发现
- eBPF 集成测试需要特殊环境支持
- 建议采用渐进式方案：先实现基础测试，再扩展到完整测试
